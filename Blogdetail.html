Hugo's Go-To Stack

Ly Kimhab
May 15, 2025

As a software engineer, choosing the right technology stack is crucial for building scalable, maintainable, and efficient applications. In this post, I’ll share my go-to stack, which I’ve refined over years of working on various projects, from web apps to backend services.

Why This Stack?

My stack is designed for rapid development, performance, and ease of scaling. It leverages modern tools and frameworks that align with my focus on quality and reliability.

  • Go: For backend services, Go’s simplicity, concurrency model (goroutines), and fast compilation make it ideal for high-performance APIs.
  • React: For frontends, React’s component-based architecture and ecosystem allow me to build dynamic, responsive UIs.
  • PostgreSQL: As a relational database, PostgreSQL offers robustness, SQL standards, and advanced features like JSONB for flexibility.
  • Docker: Containerization with Docker ensures consistent environments across development, testing, and production.

How I Use It

A typical project starts with a Go-based API serving REST or GraphQL endpoints. React handles the frontend, communicating with the backend via Axios or Apollo Client. PostgreSQL stores structured data, while Docker streamlines deployment to AWS or other cloud platforms.

This stack has powered everything from e-commerce platforms to open-source tools I’ve contributed to on GitHub. Its flexibility allows me to adapt to different project requirements while maintaining a lean codebase.

Conclusion

My go-to stack isn’t set in stone—it evolves with new tools and trends. However, Go, React, PostgreSQL, and Docker provide a solid foundation for most of my work. If you’re curious about specific tools or want to discuss your stack, reach out via email or check out my GitHub!
